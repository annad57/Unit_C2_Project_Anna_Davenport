Lesson 7- File Paths

HTML code often contains files that are on the server. To refer to them, we use file paths. There are two kinds: Relative and absolute. Files are organized in a computer or server based on a hierarchy. One root file that expands into sub-files, which can have more and more files in them.

To find a file, you must direct up or down on the hierarchy and left or right to the specific file. In one group, you can refer to files just by their name. If it is in another group, you must specifcy to direct to that group, whether that mean moving to the last common shared group and back down, and then the file inside it.

To move up a level (closer to root), you use "../". To move down a level, use "/". To move across a level, also use "/", or just put its name.

(Relative file paths used)
  • To go from F7 to F9, write "F9".
  • To go from F8 to F5, write "../../D13/F5".
  • To go from D1 to F2, write "/D11/F2".

Relative file paths

Relative file paths are used to navigate a location in the current directory, and are easy to read long as the directory itself is easy to read. Most HTML code use relative file paths, so it is good to practice.

Absolute file paths

Absolute file paths are directions to a file from the root directory. In other words, their file path writes the location of the file from the start of the directory. It is best to use them when referring to files that exist outside of your current computer or server.
