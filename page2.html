Lesson 2- Semantic Tagging

Semantic tags in HTML are tags that show their purpose to someone reading the code. Without tags that explicitly state their function, code can quickly become messy and unreadable. An example of a non-semantic tag is <div>. This tag creates divisions in a page, similar to how the semantic tags <main>, <header>, and <article> do. However, as seen, semantic tags give more information about their content and make it easier to structure code that's readable. Semantic tags make up most code and tags commonly used, and are usually recognized by if their name is relevant and specific to its content.

Example

<! DOCTYPE html>
<html>
    <head>
    </head>
    <body>
        <article>
            <h2>HTML</h2>
            <p>HTML is a coding language that uses tags to establish parts of a site.</p>
    </body>
</html>

Notice how the article tag is on its own and provides a specific topic- a crawler knows that it is going to be writing about a specific topic from the tag, not the content.
